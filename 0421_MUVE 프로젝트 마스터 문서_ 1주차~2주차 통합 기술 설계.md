## 프로젝트 개요

### 비전

MUVE는 **음악을 소리 없이 감각으로 경험하게 하는 혁신적 플랫폼**입니다. 청각장애인, 저시력 사용자, 감각 과민증 사용자 등 다양한 신체 조건을 가진 사람들이 음악의 감정과 리듬을 진동과 색상으로 느낄 수 있도록 설계되었습니다.

### 핵심 가치

**포용성(Inclusivity)**: 모든 사람이 음악을 즐길 수 있는 환경 조성  
**기술 혁신**: AI 음성 분석과 실시간 감각 효과 동기화  
**사용자 중심**: 개인의 감각 특성에 맞춘 맞춤형 설정  
**확장성**: 새로운 곡과 장르의 무한 추가 가능

## 제네릭 매핑 엔진 설계

### 3.1. 핵심 설계 원칙

#### 통일성 (Uniformity)

모든 곡이 동일한 JSON 구조를 따릅니다. Flutter 앱은 하나의 파서로 모든 곡을 처리할 수 있으며, 새로운 곡 추가 시 앱 코드 수정이 불필요합니다.

```
곡 추가 프로세스:
1. 새 음원 업로드
2. Librosa로 분석 (BPM, 에너지, 주파수)
3. 감각 매핑 규칙 적용
4. JSON 파일 생성
5. 앱에 JSON 파일만 추가 (코드 수정 없음)
```

#### 확장성 (Extensibility)

새로운 감각 효과를 추가할 때 기존 구조를 유지하면서 확장 가능합니다.

```json
// 현재 구조
{
  "haptic": { "intensity": 0.5, "type": "impact" },
  "visual": { "color": "#FFD700", "effect": "pulse" }
}

// 향후 확장 (온도, 질감 추가)
{
  "haptic": { "intensity": 0.5, "type": "impact" },
  "visual": { "color": "#FFD700", "effect": "pulse" },
  "thermal": { "temperature": 35, "sensation": "warm" },
  "tactile": { "texture": "smooth", "roughness": 0.2 }
}
```

#### 정규화 (Normalization)

모든 수치 값이 0.0~1.0 범위로 정규화되어 있어, 사용자 설정에 따른 스케일링이 용이합니다.

```
사용자가 진동 강도를 50%로 설정:
adjusted_intensity = original_intensity × 0.5

예: original_intensity = 0.8
    adjusted_intensity = 0.8 × 0.5 = 0.4
```

#### 메타데이터 포함 (Metadata Inclusion)

각 곡의 특성(BPM, 장르, 길이)을 명시하여 동적 프리셋 적용이 가능합니다.

```json
{
  "metadata": {
    "title": "귓속에맴도는멜로디",
    "genre": "indie",
    "bpm": 76.00,
    "duration_seconds": 269.36
  }
}
```

### 3.2. JSON 스키마 정의

```json
{
  "metadata": {
    "filename": "string",
    "title": "string",
    "genre": "string (indie|fusion_korean|classical)",
    "duration_seconds": "number",
    "bpm": "number",
    "sample_rate": "number",
    "analysis_version": "string (e.g., 2.0)"
  },
  "statistics": {
    "avg_energy": "number (0.0~1.0)",
    "max_energy": "number (0.0~1.0)",
    "min_energy": "number (0.0~1.0)",
    "avg_bass": "number (0.0~1.0)",
    "max_bass": "number (0.0~1.0)",
    "avg_treble": "number (0.0~1.0)"
  },
  "mapping_rules": {
    "genre": "string",
    "haptic": {
      "algorithm": "weighted_energy_bass",
      "intensity_formula": "(energy × weight_energy) + (bass × weight_bass)",
      "type_thresholds": {
        "impact": 0.7,
        "continuous": 0.3,
        "soft": 0.0
      }
    },
    "visual": {
      "algorithm": "energy_based_color_transition",
      "color_palette": {
        "low_energy": ["#90EE90", "#E6E6FA"],
        "mid_energy": ["#87CEEB", "#FFA500"],
        "high_energy": ["#FFD700", "#FFFFFF"]
      },
      "effect_types": ["diffuse", "pulse", "explosion"]
    }
  },
  "data": [
    {
      "time_ms": "number",
      "energy": "number (0.0~1.0)",
      "bass": "number (0.0~1.0)",
      "mid": "number (0.0~1.0)",
      "treble": "number (0.0~1.0)",
      "haptic": {
        "intensity": "number (0.0~1.0)",
        "type": "string (impact|continuous|soft)"
      },
      "visual": {
        "color": "string (hex color code)",
        "effect": "string (diffuse|pulse|explosion)",
        "brightness": "number (0.0~1.0)"
      }
    }
  ]
}
```

### 3.3. 데이터 크기 최적화

| 항목 | 크기 |
|---|---|
| 메타데이터 | ~1KB |
| 통계 정보 | ~0.5KB |
| 매핑 규칙 | ~2KB |
| 데이터 (0.1초 단위) | ~3~5MB (곡 길이에 따라) |
| **전체 (약 4분 곡)** | **~5~7MB** |

**최적화 전략**:
- 0.1초 단위 데이터로 충분한 정밀도 확보
- 변화량 기반 샘플링으로 불필요한 데이터 제거 가능
- gzip 압축으로 50~60% 크기 감소

## 사용자 맞춤형 설정 시스템

### 5.1. 진동 강도 조절 (0% ~ 200%)

사용자는 슬라이더를 통해 진동 강도를 조절할 수 있습니다.

| 강도 | 설정값 | 사용 사례 | 설명 |
|---|---|---|---|
| 매우 약함 | 25% | 감각 과민증 사용자 | 미세한 진동만 전달 |
| 약함 | 50% | 일반 사용자 (민감) | 부드러운 진동 |
| 보통 | 100% | 일반 사용자 | 원본 데이터 그대로 |
| 강함 | 150% | 일반 사용자 (둔감) | 강화된 진동 |
| 매우 강함 | 200% | 청각장애인 사용자 | 극대화된 진동 |

**계산식**: `adjusted_intensity = original_intensity × (haptic_strength_setting / 100)`

### 5.2. 색상 대비 조절 (일반 모드 / 고대비 모드)

저시력 사용자나 색각이상 사용자를 위해 고대비 모드를 제공합니다.

| 모드 | 채도 변화 | 명도 변화 | 대상 사용자 |
|---|---|---|---|
| 일반 모드 | 기본 | 기본 | 일반 사용자 |
| 고대비 모드 | +30% | -20% | 저시력, 색각이상 사용자 |

**색상 변환 예시**:

| 원본 색상 | 일반 모드 | 고대비 모드 |
|---|---|---|
| #E6E6FA (연보라) | #E6E6FA | #9370DB (진보라) |
| #87CEEB (하늘색) | #87CEEB | #0099FF (진하늘) |
| #FFA500 (주황색) | #FFA500 | #FF6600 (진주황) |
| #FFD700 (금색) | #FFD700 | #FFB300 (진금) |

### 5.3. 자극 민감도 프리셋 (4가지)

| 프리셋 | 진동 강도 | 시각 밝기 | 사용 사례 | 대상 사용자 |
|---|---|---|---|---|
| 명상 모드 | 50% | 70% | 휴식, 명상 | 감각 과민증 사용자 |
| 일반 모드 | 100% | 100% | 일상적 사용 | 일반 사용자 |
| 활동 모드 | 150% | 120% | 운동, 활동 | 활동적인 사용자 |
| 청각장애인 모드 | 200% | 150% | 청각장애 사용자 | 청각장애인 |

### 5.4. 설정 저장 및 복원

사용자의 맞춤형 설정을 로컬 저장소(SharedPreferences)에 저장하여, 앱을 재시작해도 설정이 유지됩니다.

```dart
// Flutter 저장 로직 예시
class UserSettingsManager {
  static const String HAPTIC_STRENGTH_KEY = 'haptic_strength';
  static const String COLOR_CONTRAST_KEY = 'color_contrast';
  static const String SENSITIVITY_PRESET_KEY = 'sensitivity_preset';
  
  Future<void> saveSettings(UserSettings settings) async {
    final prefs = await SharedPreferences.getInstance();
    await prefs.setDouble(HAPTIC_STRENGTH_KEY, settings.hapticStrength);
    await prefs.setBool(COLOR_CONTRAST_KEY, settings.colorContrastEnabled);
    await prefs.setString(SENSITIVITY_PRESET_KEY, settings.sensitivityPreset);
  }
  
  Future<UserSettings> loadSettings() async {
    final prefs = await SharedPreferences.getInstance();
    return UserSettings(
      hapticStrength: prefs.getDouble(HAPTIC_STRENGTH_KEY) ?? 100.0,
      colorContrastEnabled: prefs.getBool(COLOR_CONTRAST_KEY) ?? false,
      sensitivityPreset: prefs.getString(SENSITIVITY_PRESET_KEY) ?? 'normal'
    );
  }
}
```

### 6.2. 저지연(Low-Latency) 구현 전략

#### 프레임 기반 업데이트

- 0.1초 단위의 JSON 데이터를 기반으로 10ms 주기로 하드웨어를 업데이트
- 사용자가 음악과 감각 효과의 불일치를 인지하지 못하도록 함

#### 버퍼링 전략

- 다음 5개 프레임(0.5초)의 데이터를 미리 로드
- 메모리 접근 지연을 최소화하고 부드러운 재생 보장

#### 하드웨어 최적화

- 진동 모터와 화면 업데이트를 별도 스레드에서 처리
- UI 블로킹을 방지하고 반응성 향상

```dart
// 저지연 구현 예시
class HapticVisualSyncManager {
  late Timer _updateTimer;
  Queue<HapticFrame> _hapticBuffer = Queue();
  Queue<VisualFrame> _visualBuffer = Queue();
  
  void startSync(String jsonPath) {
    _loadNextFrames(5); // 다음 5개 프레임 미리 로드
    
    _updateTimer = Timer.periodic(Duration(milliseconds: 10), (_) {
      if (_hapticBuffer.isNotEmpty && _visualBuffer.isNotEmpty) {
        HapticFrame haptic = _hapticBuffer.removeFirst();
        VisualFrame visual = _visualBuffer.removeFirst();
        
        // 별도 스레드에서 처리
        _applyHapticEffect(haptic);
        _updateVisualEffect(visual);
        
        // 다음 프레임 로드
        if (_hapticBuffer.length < 5) {
          _loadNextFrames(1);
        }
      }
    });
  }
  
  void _applyHapticEffect(HapticFrame frame) {
    // 진동 모터 제어
    switch(frame.haptic.type) {
      case 'impact':
        HapticFeedback.heavyImpact();
        break;
      case 'continuous':
        Vibration.vibrate(duration: (100 * frame.haptic.intensity).toInt());
        break;
      case 'soft':
        HapticFeedback.lightImpact();
        break;
    }
  }
  
  void _updateVisualEffect(VisualFrame frame) {
    // 화면 색상 및 효과 업데이트
    setState(() {
      currentColor = Color(int.parse(frame.visual.color.replaceFirst('#', '0xff')));
      currentBrightness = frame.visual.brightness;
      currentEffect = frame.visual.effect;
    });
  }
}
```

### 6.3. 배터리 및 발열 최적화

#### 진동 호출 빈도 조절

**변화량 기반 샘플링**: 진동 강도의 변화량이 0.05 미만인 구간에서는 업데이트 주기를 0.2초로 늘림

**에너지 레벨 기반 조절**: 에너지가 0.1 이하인 구간에서는 진동을 완전히 비활성화

### 기술적 하이라이트

**제네릭 매핑 엔진**: 새로운 곡을 추가할 때 앱 코드 수정 없이 JSON 파일만 교체하면 작동합니다.

**중앙값 기반 프리셋**: 3개 곡의 통계 데이터를 기반으로 자동 프리셋을 생성하여, 새로운 곡이 추가될 때 즉시 적용 가능합니다.

**다층 접근성**: 진동 강도(0~200%), 색상 대비, 4가지 감각 프리셋으로 청각장애인, 저시력 사용자, 감각 과민증 사용자 등 다양한 사용자를 포용합니다.

**저지연 구현**: 0.1초 단위의 정밀한 데이터와 10ms 주기의 하드웨어 업데이트로 음악과 감각 효과의 완벽한 동기화를 보장합니다.
